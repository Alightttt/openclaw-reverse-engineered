01. What is OpenClaw Actually?

OpenClaw is NOT a model. It's NOT intelligence. It's NOT consciousness.

AGENT = Session + Skills + Tools + Model + Workspace

Without these 5 things, there's no "agent" - just an API endpoint.

🔑 KEY INSIGHT

The "personality" you experience comes from SKILLS (loaded into system prompt).
The "intelligence" comes from TOOLS + MODEL.
The "memory" comes from SESSION history.
The "agency" comes from the WRAPPER infrastructure.
The model just outputs text - nothing more.

05. What Actually Runs (Process Model)

A single long-lived Gateway process containing:

  • WebSocket Server (ws://127.0.0.1:18789)
  • Session Manager (SQLite storage)
  • Skills Loader (file watcher)
  • Tool Registry
  • Agent Executor
  • Channel Handlers (Baileys, grammY, etc)
  • Node Manager
  • Event Bus

07. The Complete 10-Step Pipeline
STEP 01
MESSAGE ARRIVES
Channel webhook → Channel Handler → sessionKey mapping
STEP 02
SESSION RESOLUTION
Look up or create session in SQLite DB
STEP 03
QUEUE
Serialize per session lane, prevent races
STEP 04
WORKSPACE
Resolve path, acquire write lock
STEP 05
SKILLS
Load skills → inject into system prompt
STEP 06
BUILD PROMPT
system + history + tools → API request
STEP 07
MODEL
Call Anthropic/OpenAI API
STEP 08
TOOLS
If tool_call → execute → inject result → loop
STEP 09
FORMAT
Extract text, remove NO_REPLY, dedupe
STEP 10
RESPONSE
Channel sendMessage → User sees reply
